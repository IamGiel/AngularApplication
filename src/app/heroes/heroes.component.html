Data Binding

Below is an art work of data binding, where components handles functionality and controls the DOM. Two-way binding also is at work as both DOM and Component talks to each other and creates changes as user interact with the page. This is demonstrated by the input field.

The buttons are controlling the images using property binding, another one way binding. The button tag uses an event listener (on click) and calls a function declared inside the component.ts file.

(click) Handlers are implemented to listen to changes and are defined with a function that hides the other button via boolean. Moreover, we have used structural directives *ngIf (for switching a boolean from true to false and vice versa) and *ngFor (looping through array defined in the component).

The input field is using `[(ngModel)]="variable"` then the variable is used in the component.ts to print the user input in li tag.

The list of traits for SquidWord is done through `service` thats sole purpose is to list traits. To do this, first create a service file using CLI or manually. Then, in the service file, define your class that will have a method that sole purpose is to have an array of values. Next, in the app.module file, you need to import this and declare it in the provider metadata. You can have multiple providers. So, place this class name inside an array when declaring it as a provider. Go back to the component files that need access to this service. Import it on the top of the file. Then, define the variable that will instantiate the class inside ngOnInit() life-cycle hook.

The font color here, is changed using attribute binding [ngStyle]. SpongeBobs "funny" trait is changed to yellow using [ngClass] for bootstrap (text-warning).

There are two types of directives. Structural and attribute directives.
Structural directives are ones that changes the structure of the DOM (e.g adds or changes texts, images, delete divs, list values from an array etc..). `*ngIf="variable"` is an example defined in the DOM and the variable manipulated through javascript logic done in the component. `*ngFor="let var of vars"` where var is variable and vars = an array of values, allows for iterating through values in an array and dsiplaying it in the DOM.
